## 概览

本决策树提供场景化的问题导航，帮助用户根据具体情况选择正确的分析路径和决策方向。

**核心原则**:
- 从用户实际问题出发，而非从理论出发
- 每个决策节点对应具体行为或场景
- 提供可操作的下一步行动

**适用场景**:
- 不确定如何开始分析
- 面对复杂情况不知道选哪个工具
- 需要快速定位问题类型

## 决策树结构

### 起始节点：你的核心问题是什么？

```
[开始]
  │
  ├─ A1: 新认识伴侣，想判断类型 → [路径A: 初期识别]
  ├─ A2: 现有关系，想评估健康度 → [路径B: 现有关系评估]
  ├─ A3: 伴侣有异常行为，担心是伪装 → [路径C: 伪装验证]
  ├─ A4: 想在多个伴侣中选择 → [路径D: 对比决策]
  ├─ A5: 关系遇到问题，不知道该走该留 → [路径E: 危机决策]
  └─ A6: 伴侣行为复杂，不知道是什么类型 → [路径F: 混合特征分析]
```

---

### 路径A：初期识别（新认识伴侣）

**A1.1: 认识多久了？**
- ≤1个月 → [警告] 过早判断，观察期，使用行为信号快速扫描
- 1-3个月 → 进入A1.2
- 3-6个月 → 进入A1.2（已过新鲜期，更可靠）
- ≥6个月 → 转到路径B（现有关系评估）

**A1.2: 在哪些场景下观察过她？**
- 只在约会场景 → [警告] 缺乏压力场景数据，建议主动创造压力测试
- 日常场景+约会场景 → 进入A1.3
- 日常场景+约会场景+压力场景 → 进入A1.3

**A1.3: 是否出现以下信号（选多个继续）？**
- 信号A：加班晚归时抱怨陪伴时间 → [倾向：丫鬟型] → 使用丫鬟型行为信号对照
- 信号B：永远展示完美形象，不知道真实状态 → [倾向：妃子型] → 使用妃子型行为信号对照
- 信号C：遇到困难时一起想办法解决问题 → [倾向：皇后型] → 使用皇后型行为信号对照
- 信号D：精致但自我驱动，行为一致 → [倾向：皇后型] → 误判点验证
- 信号E：阻拦你做有风险的决定 → [倾向：丫鬟型] → 验误判点验证

**A1.4: 初步判断结果**
- 明显倾向某一类型 → 对应章节深入学习
- 信号混合或矛盾 → 转到路径F（混合特征分析）
- 不确定 → 创建行为观察清单，记录3个月后重新评估

---

### 路径B：现有关系评估

**B1.1: 关系持续多久了？**
- <6个月 → [警告] 仍可能在伪装期，参考路径C
- 6个月-1年 → 进入B1.2
- 1-3年 → 进入B1.2
- >3年 → 进入B1.2（长期关系，更稳定）

**B1.2: 关系整体状态？**
- 感觉良好，想确认健康度 → 使用结构化评估模型
- 感觉有消耗，不知道问题在哪 → 进入B1.3
- 有明显问题，考虑止损 → 转到路径E（危机决策）

**B1.3: 消耗感来源（选多个）？**
- 情绪消耗：她情绪不稳定，拉低你 → [维度：情绪稳定性] → 使用评估模型重点评估
- 发展消耗：阻拦你成长/事业 → [维度：成长支持度] → 使用评估模型重点评估
- 精力消耗：需要不断证明价值/安全感 → [维度：自我价值锚点] → 使用评估模型重点评估
- 不确定消耗来源 → 使用风险预警系统扫描

**B1.4: 评估结果解读**
- 得分≤4分 → 转到路径E（危机决策）
- 得分4-6分 → 使用动态跟踪框架观察改进
- 得分6-8分 → 持续优化，参考成长支持建议
- 得分≥8分 → 优质关系，珍惜并深化

---

### 路径C：伪装验证

**C1.1: 为什么怀疑是伪装？**
- 表现太好，不像真的 → 进入C1.2
- 前后不一致 → 进入C1.3
- 关系稳定后突然变化 → 进入C1.4

**C1.2: 表现太好验证**
- 创造压力测试：故意暴露弱点、制造小危机、暂停物质投入
- 观察3个月变化
- 若在压力下表现一致 → 可能是真的皇后型
- 若在压力下原形毕露 → 妃子型伪装，转到路径E

**C1.3: 前后不一致验证**
- 识别变化时间点：关系稳定后？你事业上升后？你遇到困难后？
- 对照变化前后的行为模式
- 若变化后更像丫鬟型/妃子型 → 伪装期结束，真实格局暴露
- 使用动态跟踪框架记录变化趋势

**C1.4: 关系稳定后突然变化验证**
- 识别变化触发事件
- 判断是阶段性不安全感（可调整）还是深层格局限制（难调整）
- 使用评估模型重新打分
- 参考决策支持矩阵判断调整可能性

---

### 路径D：对比决策（多伴侣选择）

**D1.1: 有几个候选伴侣？**
- 2个 → 进入D1.2
- 3个及以上 → 进入D1.3

**D1.2: 二选一对比**
- 对每个伴侣使用结构化评估模型
- 对比六个维度得分
- 关注：
  - 综合得分差值（>2分才有明显优势）
  - 情绪稳定性+成长支持度（核心维度）
  - 长期匹配度（关键维度）
- 决策：选择综合得分更高且核心维度无短板者

**D1.3: 多选一决策**
- 对所有伴侣使用结构化评估模型
- 排除风险等级（得分≤4分）
- 按综合得分排序
- 考察前2名的具体案例
- 使用决策支持矩阵做最终决策

---

### 路径E：危机决策（该走该留）

**E1.1: 当前关系状态？**
- 已经准备离开 → 进入E1.2
- 还在犹豫，但消耗严重 → 进入E1.3

**E1.2: 准备离开决策**
- 使用结构化评估模型确认风险等级
- 使用风险预警系统确认危险信号
- 确认已无调整空间：
  - 沟通无效
  - 对方无成长意愿
  - 格局限制而非阶段问题
- 参考决策支持矩阵中的止损策略

**E1.3: 犹豫决策**
- 使用结构化评估模型识别问题维度
- 识别是能力问题、意愿问题还是格局问题
- 使用决策支持矩阵评估调整可能性
- 制定3个月观察计划：
  - 明确沟通问题
  - 设定边界和期望
  - 使用动态跟踪框架记录
- 3个月后重新评估

---

### 路径F：混合特征分析

**F1.1: 哪些特征混合了？（选多个）**
- 丫鬟型特征+妃子型表象 → 使用混合特征识别器-类型A
- 皇后型特征+偶尔丫鬟型表现 → 使用混合特征识别器-类型B
- 多个类型特征交替出现 → 使用混合特征识别器-类型C

**F1.2: 混合特征识别**
- 使用混合特征识别器进行深度分析
- 识别主特征和次特征
- 分析切换触发条件
- 预测长期趋势

**F1.3: 混合型伴侣决策**
- 使用评估模型量化主特征影响
- 评估改进可能性
- 参考决策支持矩阵

## 节点定义

### 警告节点
含义：当前数据不足或判断风险过高，建议补充数据或调整评估策略

### 倾向节点
含义：初步判断倾向某一类型，但需要进一步验证

### 验证节点
含义：已有初步结论，需要通过特定方法验证

### 决策节点
含义：需要基于多个维度做最终决策

## 使用方法

### 基本流程
1. 从起始节点开始，回答"你的核心问题是什么？"
2. 根据回答进入对应路径
3. 依次回答路径上的问题
4. 在每个决策节点查看对应建议
5. 根据建议执行下一步行动

### 快速导航
如果你已经知道自己的需求，可以直接跳转到对应路径：
- 新伴侣判断 → 路径A
- 现有关系评估 → 路径B
- 伪装验证 → 路径C
- 多伴侣对比 → 路径D
- 危机决策 → 路径E
- 混合特征 → 路径F

## 场景示例

### 示例1：新认识3个月，想判断类型

**路径导航**:
1. 起始节点 → A1: 新认识伴侣，想判断类型 → [路径A: 初期识别]
2. A1.1: 认识多久了？ → 1-3个月 → 进入A1.2
3. A1.2: 在哪些场景下观察过她？ → 日常场景+约会场景 → 进入A1.3
4. A1.3: 是否出现以下信号？ → 信号B：永远展示完美形象，不知道真实状态 → [倾向：妃子型]
5. 建议行动：使用妃子型行为信号对照，同时创建压力测试验证

**执行建议**:
- 主动创造压力场景：暴露弱点、制造小危机、暂停物质投入
- 观察她在压力下的反应
- 3个月后重新评估

### 示例2：在一起2年，感觉有消耗

**路径导航**:
1. 起始节点 → A2: 现有关系，想评估健康度 → [路径B: 现有关系评估]
2. B1.1: 关系持续多久了？ → 1-3年 → 进入B1.2
3. B1.2: 关系整体状态？ → 感觉有消耗，不知道问题在哪 → 进入B1.3
4. B1.3: 消耗感来源？ → 发展消耗：阻拦你成长/事业 → [维度：成长支持度]
5. 建议行动：使用结构化评估模型重点评估成长支持度维度

**执行建议**:
- 使用评估模型打分
- 得分≤3分 → 转到路径E（危机决策）
- 得分3-5分 → 制定沟通计划，设定边界
- 得分5-7分 → 观察改进，使用动态跟踪框架

### 示例3：伴侣表现太好，担心是伪装

**路径导航**:
1. 起始节点 → A3: 伴侣有异常行为，担心是伪装 → [路径C: 伪装验证]
2. C1.1: 为什么怀疑是伪装？ → 表现太好，不像真的 → 进入C1.2
3. 建议行动：创造压力测试，观察3个月变化

**执行建议**:
- 故意暴露弱点：分享失败经历、展示脆弱
- 制造小危机：暂停物质投入、减少见面频率
- 观察她在压力下的反应：
  - 若表现一致 → 可能是真的皇后型
  - 若原形毕露 → 妃子型伪装

## 注意事项
- 决策树提供的是导航，不是替代思考
- 每个建议都需要结合具体情境执行
- 伪装验证需要时间，避免过早下结论
- 混合特征需要长期观察，避免简单贴标签
- 最终决策权在你，决策树只是辅助工具
